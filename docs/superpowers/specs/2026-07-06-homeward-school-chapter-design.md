# 집에 가고 싶다 - 1챕터 디자인

## 개요

`집에 가고 싶다`는 일상 공간을 탈출 던전처럼 과장해 표현하는 Godot 기반 2D 액션 러너 게임이다. 전체 게임은 회사, 학교, 알바, 퇴근길 같은 여러 챕터로 확장할 수 있게 설계하되, 첫 구현은 학교를 배경으로 한 1챕터에 집중한다.

1챕터의 목표는 "학교에서 집으로 탈출한다"는 감정을 빠르게 플레이로 전달하는 것이다. 플레이어는 수업과 종례가 끝나지 않는 학교에서 벗어나기 위해 자동 진행 러너 구간을 통과하고, 마지막 관문에서 자유 이동 액션 보스전을 치른다.

## 레퍼런스 방향

핵심 레퍼런스는 CrazyGames의 `Electron Dash`다. 이 게임처럼 플레이어가 앞으로 자동 진행하면서 구멍, 레이저, 불안정한 바닥을 피하고, 진행할수록 속도와 긴장이 올라가는 감각을 목표로 한다.

단, 1챕터는 Godot 2D 기반으로 만든다. 실제 3D 튜브를 그대로 구현하지 않고, 2D 레인 러너 위에 배경 회전, 타일 스케일 변화, 카메라 흔들림, 원근감 있는 바닥 배치로 2.5D 터널 질감을 낸다.

## 1챕터 콘셉트

1챕터 제목은 `종례를 뚫고 집으로`로 확정한다.

학교 복도는 플레이어의 "집에 가고 싶다" 욕망 때문에 차원처럼 뒤틀린다. 처음에는 현실적인 교실과 복도지만, 시간이 지날수록 책상, 시험지, 종례 방송, 분필, 출석부가 과장된 함정과 공격 패턴으로 변한다.

챕터 흐름:

1. 교실 러너 구간
2. 복도 터널 러너 구간
3. 계단 및 현관 러너 구간
4. 교문 앞 보스전
5. 탈출 컷신

## 핵심 플레이

러너 구간에서는 주인공이 자동으로 앞으로 달린다. 플레이어는 좌우 레인 이동, 점프, 이단점프, 숙이기 또는 빠른 낙하를 사용해 장애물을 피한다.

러너 구간 조작:

- `A/D` 또는 `Left/Right`: 좌우 레인 이동
- `Space` 또는 `W`: 점프
- 공중에서 `Space` 또는 `W`: 이단점프
- `S` 또는 `Down`: 숙이기, 빠른 낙하

레인 이동은 단순히 캐릭터 위치만 바꾸는 것이 아니라, 화면 속 복도나 바닥이 살짝 회전하는 것처럼 연출한다. Electron Dash의 "통로가 함께 움직이는" 감각을 2D로 번역한 장치다.

## 장애물

첫 구현 장애물은 명확히 읽히고 타이밍 학습이 쉬운 것부터 만든다.

- 빠지는 바닥 타일: 밟기 직전 색이 바뀌고, 잠시 후 사라진다.
- 시험지 레이저: 화면 가로 또는 세로 방향으로 지나가는 종이 빔이다.
- 책상 더미: 점프로 넘는다.
- 낮은 종례 방송 충격파: 숙이기로 피한다.
- 분필 탄막: 정해진 레인으로 날아오며 좌우 이동으로 피한다.
- 집 열쇠 아이템: 욕망 게이지를 채운다.

첫 버전에서는 장애물을 수동 배치한 패턴 목록에서 순서대로 스폰한다. 이후 챕터 확장 단계에서 랜덤 패턴이나 난이도 곡선을 추가한다.

## 욕망 게이지

UI에는 `집에 가고 싶다` 게이지가 표시된다. 장애물을 아슬아슬하게 피하거나, 집 열쇠와 간식 같은 아이템을 먹으면 게이지가 오른다.

게이지가 가득 차면 `폭주 모드`가 발동된다.

러너 구간 폭주 모드:

- 일정 시간 속도가 상승한다.
- 작은 장애물을 부수며 지나갈 수 있다.
- 화면 색감, 카메라 흔들림, 배경 왜곡이 강해진다.
- 위험도도 함께 올라가도록 장애물 예고 시간이 짧아진다.

보스전 폭주 모드:

- 필살기 `귀가 본능 폭발`을 사용할 수 있다.
- 보스 체력에 큰 피해를 준다.
- 사용 후 게이지가 비워진다.

## 보스전

최종 보스는 `담임 선생님: 종례의 수호자`다.

러너 구간 끝에서 교문 앞에 도착하면 자동 달리기가 멈추고, 보스전용 2D 액션 구간으로 전환된다. 이 구간에서는 좌우 자유 이동, 점프, 이단점프, 숙이기, 공격, 회피 대시를 사용한다.

보스전 조작:

- `A/D` 또는 `Left/Right`: 좌우 이동
- `Space` 또는 `W`: 점프
- 공중에서 `Space` 또는 `W`: 이단점프
- `S` 또는 `Down`: 숙이기
- `Z`: 공격
- `Shift`: 짧은 회피 대시

보스 패턴:

- 분필 투척: 직선 탄환 패턴이다.
- 출석부 내려찍기: 바닥 충격파를 만든다.
- 시험지 탄막: 여러 레인 또는 높이로 날아온다.
- 종례 멘트 충격파: 숙이기 또는 점프로 피하는 큰 파동이다.

플레이어는 패턴을 피하다가 보스가 지친 순간에 공격한다. 보스 체력이 0이 되면 교문이 열리고 짧은 탈출 컷신으로 이어진다.

## 컷신

컷신은 고화질 연출을 목표로 하되, 첫 구현에서는 Godot 안에서 재생 가능한 2D 연출로 만든다. 시작 컷신은 교실에서 시계를 바라보는 짧은 장면, 엔딩 컷신은 교문 밖 빛을 향해 달려나가는 장면이다.

첫 버전에서는 영상 파일보다 Godot 씬 기반 컷신을 우선한다. 배경 이미지, 캐릭터 스프라이트, 카메라 이동, 텍스트 박스, 화면 전환을 조합한다. 이후 고화질 일러스트나 렌더 영상으로 교체할 수 있게 컷신 데이터를 별도 씬으로 분리한다.

## Godot 구조

대상 엔진은 Godot 4.x다. 프로젝트는 2D 기반으로 만들고, 터널 러너 감각은 2D 노드와 셰이더/카메라 연출로 표현한다.

권장 폴더 구조:

- `scenes/`: 메인 메뉴, 챕터, 러너, 보스전, 컷신 씬
- `scripts/`: 플레이어, 러너 컨트롤러, 장애물, 보스, UI 스크립트
- `assets/art/`: 캐릭터, 배경, 장애물, UI 이미지
- `assets/audio/`: 효과음과 음악
- `assets/cutscenes/`: 컷신용 이미지와 데이터
- `resources/`: 장애물 패턴, 챕터 설정, 보스 패턴 데이터

핵심 컴포넌트:

- `GameRoot`: 전체 상태 전환을 관리한다.
- `RunnerStage`: 자동 진행, 레인 이동, 장애물 스폰, 진행 거리 계산을 담당한다.
- `RunnerPlayer`: 러너 조작, 점프, 이단점프, 숙이기, 피격을 담당한다.
- `LaneTunnelView`: 2D 레인 좌표를 터널처럼 보이게 배치하고 카메라 연출을 담당한다.
- `ObstacleSpawner`: 수동 패턴 데이터를 읽고 장애물을 생성한다.
- `DesireMeter`: 욕망 게이지 값을 관리하고 UI 및 폭주 모드 이벤트를 발생시킨다.
- `BossArena`: 보스전 상태, 경계, 전투 흐름을 담당한다.
- `HomeroomBoss`: 담임 선생님 보스 패턴과 체력을 관리한다.
- `CutscenePlayer`: 씬 기반 컷신 재생을 담당한다.

## 데이터 흐름

`GameRoot`는 현재 모드를 `IntroCutscene`, `Runner`, `Boss`, `EndingCutscene` 중 하나로 관리한다. 각 모드는 끝났을 때 신호를 보내고, `GameRoot`가 다음 모드로 전환한다.

러너 구간에서 `RunnerStage`는 진행 거리와 패턴 타이밍을 관리한다. `ObstacleSpawner`는 패턴 데이터에 따라 장애물을 만들고, 플레이어 충돌은 `RunnerPlayer`가 처리한다. 욕망 게이지 변화는 `DesireMeter`로 모이며, UI는 이 값을 표시한다.

보스전에서는 `BossArena`가 플레이어와 보스를 배치한다. `HomeroomBoss`는 패턴별 신호를 내고, 플레이어 공격 또는 필살기가 보스 체력을 줄인다. 보스 격파 신호가 발생하면 `GameRoot`가 엔딩 컷신으로 전환한다.

## 실패와 재시작

러너 구간에서 체력이 0이 되면 해당 러너 구간 시작 지점에서 재시작한다. 첫 버전에서는 체크포인트를 세분화하지 않고, 챕터 플레이 시간이 짧도록 조정한다.

보스전에서 체력이 0이 되면 보스전 시작 지점에서 재시작한다. 욕망 게이지는 절반만 유지해 재도전이 너무 답답하지 않게 한다.

## 테스트 기준

첫 구현 완료 기준:

- Godot 프로젝트가 정상 실행된다.
- 메인 씬에서 1챕터가 시작된다.
- 러너 구간에서 좌우 이동, 점프, 이단점프, 숙이기가 동작한다.
- 최소 5종 장애물이 등장하고 피격 판정이 동작한다.
- 욕망 게이지가 차고 폭주 모드가 발동된다.
- 러너 구간 끝에서 보스전으로 전환된다.
- 보스가 최소 4개 패턴을 사용한다.
- 플레이어 공격과 필살기로 보스를 처치할 수 있다.
- 보스 처치 후 엔딩 컷신으로 넘어간다.

## 첫 구현 범위

첫 구현은 플레이 가능한 세로 조각을 목표로 한다. 완성된 아트와 음원보다, 조작감과 장면 전환, 장애물 판정, 보스전 루프를 우선한다.

포함:

- Godot 프로젝트 생성
- 1챕터 메인 흐름
- 2.5D 터널처럼 보이는 러너 프로토타입
- 기본 플레이어 컨트롤
- 장애물과 아이템
- 욕망 게이지와 폭주 모드
- 담임 선생님 보스전
- 간단한 시작/엔딩 컷신

제외:

- 회사, 알바, 퇴근길 챕터
- 완성형 고해상도 컷신 영상
- 저장/로드
- 난이도 선택
- 모바일 터치 UI
- 온라인 랭킹

## 확장 방향

1챕터가 안정되면 같은 구조로 회사, 알바, 퇴근길 챕터를 추가한다. 각 챕터는 러너 구간의 테마 장애물과 보스만 바꾸고, 공통 러너 시스템과 보스 아레나 시스템을 재사용한다.

장기적으로는 챕터별 컷신 품질을 높이고, Electron Dash식 무한 모드를 별도 도전 모드로 추가할 수 있다.
